Findings

01AGENTS.md / CLAUDE.md🚀 Ship-Ready
AGENTS.md declares stack and six non-obvious rules: secrets-from-env, no-content-in-logs, dead-code-as-error, idempotent mutations, warnings-as-errors, search-vs-ingest env split. CLAUDE.md mirrors run commands. Both are short and quoteable.
02Strict compiler settings👍 Solid
PaperClaw/PaperClaw/PaperClaw.csproj:7-10 sets Nullable=enable, TreatWarningsAsErrors=true, AnalysisLevel=latest-recommended, EnforceCodeStyleInBuild=true. Build is clean. No latest-all, no extra analyzer ruleset.
03Strict linter / formatter👍 Solid
dotnet format enforced via lefthook.yml:7 (--verify-no-changes) and .editorconfig. No Roslynator / SonarAnalyzer / StyleCop package wired in beyond the SDK defaults.
04Schema validation at boundaries👍 Solid
Claude tool schemas declared inline at SearchAgent.cs:22-39; classifier deserialises into typed record ClassificationResult. No System.Text.Json source-gen contract, no JSON-schema-validated boundary if Claude returns a malformed payload — it'll just throw at deserialise time.
05Logic separated from I/O🚀 Ship-Ready
Pure helpers SlugGenerator, TranscriptFormatter, ClassificationResult, plus file-backed-but-mockable ProcessedUidStore. Each has a dedicated NUnit test in PaperClaw.Tests/.
06One-command bring-up🚀 Ship-Ready
.mise.toml defines mise run setup (restore + lefthook install) and mise run check (format + build + test). Single solution = no per-folder special cases.
07Pre-commit feedback loop🚀 Ship-Ready
lefthook.yml runs gitleaks + dotnet format --verify-no-changes + dotnet build in parallel pre-commit; dotnet test pre-push. Installed via lefthook install in the setup task.
08Dead-code guardrail👍 Solid
.editorconfig:19-20 raises IDE0051/IDE0052 to warnings; combined with TreatWarningsAsErrors they fail the build. Covers unused private members only — no detector for unused public symbols, unused project references, or dead packages.
09Logs reachable from terminal🚀 Ship-Ready
Microsoft.Extensions.Logging.Console wired in Program.cs:41; level controlled via PAPERCLAW_LOG_LEVEL. Everything goes to stdout — agent-readable by default.
10Docs stay in sync with code💀 Broken
DESIGN.md, AGENTS.md, CLAUDE.md all exist and are accurate today, but nothing flags drift. lefthook.yml has no docs-touched check, no CI exists, no generated reference. Next refactor will silently desync.
Fix: add a lefthook pre-commit command that errors if files under PaperClaw/PaperClaw/*.cs are staged without a touch to DESIGN.md or AGENTS.md — or a CLI --help snapshot test under PaperClaw.Tests/.
11Agent self-tests end-to-end🚀 Ship-Ready
CLI entrypoint at Program.cs:9 exposes ingest and search <query>, results to stdout. Discoverable from AGENTS.md, CLAUDE.md, and a dedicated .claude/skills/paperclaw-search/SKILL.md. Agent can run the real CLI against a real library to verify its own change.
12Agentic review panel💀 Broken
No REVIEW.md. No /review slash command in .claude/. No script, Makefile target, or AGENTS.md instructions that spawn parallel specialist reviewers. The human is the first reviewer of every diff.
Fix: drop a .claude/commands/review.md that fans out four parallel Task agents (best-practices, C# / .NET idioms, security, maintainability) against the current branch diff, and add a REVIEW.md listing what not to flag (theoretical risks, unchanged code).
13Friction proportional to blast radius💀 Broken
No danger-zone hook, no CODEOWNERS, no named bypass env var. lefthook.yml treats every path identically. AppConfig.cs / ProcessedUidStore.cs (idempotency-critical) and .env.example can be silently changed.
Fix: add a lefthook pre-push command guarded by a glob over AppConfig.cs / ProcessedUidStore.cs / .env.example that prints a checklist (idempotency tests pass? rotated secrets?) and requires PAPERCLAW_DANGER_OK=1 to proceed.
14Tooling tuned for the agent🩹 Patchy
.gitleaksignore exists (good — clean signal). But hook failures just surface raw dotnet format / dotnet build exit codes; no remediation line tells the agent "run dotnet format PaperClaw/PaperClaw.sln to fix". mise run check exists but isn't pointed at from failure output.
Fix: wrap each lefthook command with a one-line failure echo, e.g. run: dotnet format ... --verify-no-changes --no-restore || (echo '↪ run: mise run fmt' && exit 1).

🎁 Bonus finds

  • mise run check — bundles format-check + build + test into a single verb the agent can call before declaring done. One source of truth for "is it green".
  • paperclaw-search Claude skill — wraps the search CLI as an invokable agent capability with usage guidance and result-presentation rules. The CLI becomes a tool, not a guess.
  • skills-lock.json — pins skill source + SHA256 hash. Reproducible skill installs so an agent doesn't silently pull a different version of bootstrap-project.
